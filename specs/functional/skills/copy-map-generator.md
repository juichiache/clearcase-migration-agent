# Skill: `copy_map_generator`

**Phase:** Implementation Agent
**Mode:** Read-write

---

## Purpose

Generates copy-map entries for a package in the team's existing copy-map format. This skill is the bridge between the agent's internal `PackageAssignment.json` representation and the actual copy-map files used by the build system.

> [OPEN] §3.1 — The exact copy-map file format is not yet defined. This spec will be updated when the format is confirmed. See `open-questions.md §3.1`. The worked example below uses a placeholder format.

---

## Inputs

| Input | Type | Required | Description |
|---|---|---|---|
| `packageName` | string | Yes | The package to generate a copy-map for |
| `packageAssignment` | `PackageAssignment` | Yes | Accepted assignments (status must be `"accepted"`) |
| `copyMapTemplate` | string | No | Path to an existing copy-map file for format reference |
| `outputDir` | string | Yes | Directory to write the generated copy-map file |

---

## Outputs

- A copy-map file written to `{outputDir}/{packageName}.copymap` (format TBD)
- An entry in `ImplementationLog.json` recording the generated file

---

## Preconditions

Before generating output, verify:

1. `packageAssignment.assignments` contains at least one entry for `packageName` with `status: "accepted"`
2. The package is NOT locked (`locked: false` in `ExistingPackageMap`)
3. `outputDir` is writable

Halt with error if any precondition fails.

---

## Behavior

### 1. Extract Accepted Assignments

Filter `PackageAssignment.assignments` for entries where:
- `package === packageName`
- `status === "accepted"`

### 2. Sort Files

Sort the file list deterministically (alphabetical by `filePath`) for reproducible output.

### 3. Apply Format

Format each file assignment according to the team's copy-map format.

> [OPEN] §3.1 — Replace this step with the real format when confirmed.

**Assumed placeholder format (one file per line; optional annotation after `#`):**
```
src/core/allocator.cpp
include/core/allocator.h   # public API
src/core/memory.cpp
```

### 4. Write File

Write the generated content to `{outputDir}/{packageName}.copymap`.

### 5. Log to ImplementationLog

Append to `ImplementationLog.json`:
```json
{
  "packageName": "libcore",
  "generatedFiles": [
    { "type": "copymap", "path": "copy-maps/libcore.copymap" }
  ]
}
```

---

## Error Handling

| Error Condition | Behavior |
|---|---|
| No accepted assignments for package | Halt: "No accepted assignments found for {packageName}. Ensure PackageAssignment.json is approved." |
| Package is locked | Halt: "Package {packageName} is locked and cannot be regenerated." |
| Output file already exists | Overwrite with a warning logged: "Overwriting existing copy-map for {packageName}." |
| Output directory not writable | Halt with error. |

---

## Worked Example

**Input:**
- `packageName`: `libcore`
- From `PackageAssignment.json` (accepted entries):
  - `src/core/allocator.cpp` → `libcore`
  - `src/core/memory.cpp` → `libcore`
  - `include/core/allocator.h` → `libcore`

**Generated `copy-maps/libcore.copymap`:**
```
# Generated by copy_map_generator
# Package: libcore
# Run: plan-20241215T143022Z
# DO NOT EDIT — regenerate via: @agent implement --package libcore

include/core/allocator.h
src/core/allocator.cpp
src/core/memory.cpp
```

**`ImplementationLog.json` entry:**
```json
{
  "packageName": "libcore",
  "planTaskRef": "assignment-libcore-20241215",
  "generatedFiles": [
    { "type": "copymap", "path": "copy-maps/libcore.copymap", "lineCount": 7 }
  ],
  "prUrl": null,
  "ciStatus": "pending"
}
```
